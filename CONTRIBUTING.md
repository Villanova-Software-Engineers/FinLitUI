We welcome contributions that help improve FinLit. To ensure smooth collaboration, please follow the guidelines below:

Create a new branch
Always create a new branch from main (or dev) for your changes. Avoid committing directly to the main branch.

Make focused changes
Keep pull requests small and focused. Each PR should address a single bug, feature, or improvement whenever possible.

Submit a pull request (PR)
When submitting a PR, include:

A clear summary of what the change does

The reason the change is required

If it fixes a bug, describe the bug and how it was resolved

If it introduces a new feature, explain the motivation and why it adds value to FinLit

Testing & verification

Ensure your code builds and runs successfully

Add or update tests if applicable

Verify that your changes do not break existing functionality

Code quality & style

Follow existing code style and conventions

Write clear, readable, and well-documented code

Remove unused code, logs, or commented-out sections

Commit messages

Use clear and descriptive commit messages

Example: Fix incorrect interest calculation for compound savings

Avoid vague messages like fix bug or update

Screenshots & demos (if applicable)

Include screenshots, GIFs, or screen recordings for UI changes

This helps reviewers understand the impact of your changes quickly

Review process

Be open to feedback and requested changes

Address review comments promptly

Maintainers may request revisions before merging
